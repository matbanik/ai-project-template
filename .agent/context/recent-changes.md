## Update Protocol

**When to log changes:**
1. After completing a feature or fix
2. After significant refactoring
3. After configuration changes
4. After dependency updates
5. Before ending a work session with uncommitted conceptual changes

**What to include:**
- Brief description of what changed
- Rationale (why this approach)
- Files/components affected
- Any follow-up needed

**What NOT to include:**
- Minor typo fixes
- Formatting-only changes
- WIP that will be revised

**Maintenance:**
- Roll entries to "Last Week" after 7 days
- Archive to pomera_notes monthly: `Memory/Changes/{month}-{year}`
- Keep only actionable history in this file
