## Update Protocol

**When to add issues:**
1. Bug discovered during development
2. Unexpected behavior observed
3. Design limitation identified
4. External dependency problem

**When to resolve:**
1. Issue is confirmed fixed
2. Move to "Resolved" section with solution notes
3. Keep for 30 days then archive to pomera_notes

**Severity Guide:**
| Level | Definition |
|-------|------------|
| Critical | System unusable, data loss risk |
| High | Major feature broken, no workaround |
| Medium | Feature degraded, workaround exists |
| Low | Minor inconvenience, cosmetic |
